# FRAMEWORK DE LEVANTAMIENTO DE REQUERIMIENTOS

Este conjunto de documentos permite estructurar el levantamiento de información según el tipo de servicio requerido.

---

## TIPOS DE MODELO

### 1. OUTSOURCING
Usar cuando:
- El cliente necesita un recurso
- No tiene claridad total del alcance
- Requiere apoyo continuo

---

### 2. PROYECTO ÁGIL
Usar cuando:
- El cliente tiene una idea general
- El alcance puede evolucionar
- Se trabajará por fases incrementales

---

### 3. PROYECTO CERRADO
Usar cuando:
- El cliente tiene requerimiento claro
- Se necesita precio fijo
- El alcance debe estar completamente definido

---

## RECOMENDACIONES

- Completar en reuniones con el cliente
- No asumir información no validada
- Documentar riesgos y exclusiones siempre
- Validar el resumen antes de generar propuesta

---

## FLUJO SUGERIDO

1. Completar el levantamiento
2. Validar con el cliente
3. Definir modelo de servicio
4. Generar propuesta técnica
5. Generar propuesta económica
